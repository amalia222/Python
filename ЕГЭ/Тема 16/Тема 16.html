📝Введение

В задании 16 ЕГЭ по информатике часто встречаются рекурсивно заданные функции. Требуется найти значение аргумента, при котором функция принимает заданное значение.

Типовая формулировка задачи выглядит так:

📋 Пример условия задачи:

Алгоритм вычисления значения функции F(n), где n — целое неотрицательное число, задан следующими соотношениями:

  • F(0) = 0
  • F(n) = F(n − 1) + 1, если n > 0 и n mod 3 = 2
  • F(n) = F((n − n mod 3) / 3), если n > 0 и n mod 3 < 2

Укажите наименьшее возможное n, для которого F(n) = 6.

Ответ: 728

Существует три основных подхода к решению таких задач:

  • Прямая рекурсия: перевести условие в код «один в один».
  • Рекурсия с увеличенным лимитом: если возникает RecursionError, увеличить глубину стека.
  • Мемоизация: сохранять уже вычисленные значения, чтобы не пересчитывать их снова.

🔹Шаблон решения задания рекурсией

Рекурсивная функция — это функция, которая вызывает саму себя. Чтобы она завершилась, у неё должен быть базовый случай (условие выхода).

📐 Структура рекурсивной функции:

def f(n):
    if <базовый случай>:          # условие остановки
        return <базовое значение>
    elif <условие 1>:
        return f(<меньший аргумент>) + <что-то>
    elif <условие 2>:
        return f(<ещё меньший аргумент>)

Задача:

Алгоритм вычисления функции F(n) задан соотношениями:

  • F(0) = 0
  • F(n) = F(n − 1) + 1, если n > 0 и n mod 3 = 2
  • F(n) = F((n − n mod 3) / 3), если n > 0 и n mod 3 < 2

Найти наименьшее n, при котором F(n) = 6.

💡 Подсказка: Переписываем условие задачи в код «один в один», не думая о деталях реализации.

Решение (прямая рекурсия):

💡 Способ 1: Рекурсия с if/elif
def f(n):
    if n == 0:
        return 0
    elif n % 3 == 2 and n > 0:
        return f(n - 1) + 1
    elif n % 3 < 2 and n > 0:
        return f((n - n % 3) // 3)

for i in range(1000):
    if f(i) == 6:
        print(i)  # Результат: 728
        break
💡 Способ 2: Однострочная рекурсия
def f(n):
    return 0 if n == 0 else (f(n - 1) + 1 if n % 3 == 2 else f((n - n % 3) // 3))

for i in range(1000):
    if f(i) == 6:
        print(i)  # Результат: 728
        break
⚠️ Важно: При малом диапазоне значений (до ~900) прямая рекурсия работает отлично. Проблемы начинаются при больших n, когда глубина стека вызовов превышает предел Python.

⚠️Борьба с ошибкой RecursionError

Python по умолчанию ограничивает глубину рекурсии значением 1000. Если рекурсивная функция вызывает себя глубже этого предела, возникает ошибка:

❌ Сообщение об ошибке:

RecursionError: maximum recursion depth exceeded

🛠️ Метод 1: Увеличение лимита рекурсии

Самый быстрый способ — явно увеличить предел с помощью sys.setrecursionlimit():

💡 Увеличение лимита рекурсии:
import sys
sys.setrecursionlimit(10000)  # устанавливаем лимит 10 000 вместо 1 000

def f(n):
    if n == 0:
        return 0
    elif n % 3 == 2:
        return f(n - 1) + 1
    else:
        return f((n - n % 3) // 3)

for i in range(5000):
    if f(i) == 6:
        print(i)  # Результат: 728
        break
✅ Когда помогает: Когда глубина рекурсии немного превышает 1000. Обычно значения 5000–10000 решают проблему для задач ЕГЭ.
⚠️ Ограничение: Устанавливать слишком большие значения (> 50000) опасно — можно переполнить стек операционной системы и получить сегфолт.

🛠️ Метод 2: Итеративное решение (без рекурсии)

Если увеличение лимита не помогло, или задача требует перебора очень большого диапазона, рекурсию можно заменить циклом с ручным стеком или переосмыслить алгоритм.

💡 Итеративная реализация с ручным стеком:
def f_iterative(n):
    stack = []
    additions = 0

    while n != 0:
        if n % 3 == 2:
            additions += 1
            n -= 1
        else:
            n = (n - n % 3) // 3

    return additions

for i in range(5000):
    if f_iterative(i) == 6:
        print(i)  # Результат: 728
        break
✅ Преимущество: Не зависит от лимита рекурсии, работает при любой глубине. Идеально для больших диапазонов.

🛠️ Метод 3: Декоратор @lru_cache (встроенная мемоизация)

Python предоставляет встроенный декоратор @functools.lru_cache, который автоматически кэширует результаты рекурсивной функции и уменьшает реальную глубину стека.

💡 Рекурсия с @lru_cache:
import sys
import functools

sys.setrecursionlimit(5000)

@functools.lru_cache(maxsize=None)
def f(n):
    if n == 0:
        return 0
    elif n % 3 == 2:
        return f(n - 1) + 1
    else:
        return f((n - n % 3) // 3)

for i in range(5000):
    if f(i) == 6:
        print(i)  # Результат: 728
        break
Метод Сложность реализации Ограничения Рекомендация
sys.setrecursionlimit() Минимальная (1 строка) Может упасть при очень больших значениях Первый шаг при RecursionError
Итеративный алгоритм Средняя (нужно переосмыслить логику) Нет ограничений Когда лимит не помогает
@lru_cache Минимальная (1 декоратор) Требует hashable аргументов Лучший вариант в паре с setrecursionlimit

⚡Решение через список (мемоизация рекурсии)

Мемоизация — это техника оптимизации, при которой результаты уже вычисленных вызовов функции сохраняются в словаре или списке. При повторном вызове с теми же аргументами берётся готовый результат, а не пересчитывается заново.

🧠 Идея мемоизации:

  • Без мемоизации: f(8) → f(7) → f(6) → ... (каждый раз заново)
  • С мемоизацией: f(8) → смотрим в кэш → берём готовое значение

Задача:

Та же функция F(n). Найти наименьшее n, для которого F(n) = 6.

Решение:

💡 Способ 1: Мемоизация через словарь (dict)
cache = {}  # словарь для хранения вычисленных значений

def f(n):
    if n in cache:      # если уже вычисляли — возвращаем готовое
        return cache[n]

    if n == 0:
        result = 0
    elif n % 3 == 2:
        result = f(n - 1) + 1
    else:
        result = f((n - n % 3) // 3)

    cache[n] = result   # сохраняем результат
    return result

for i in range(5000):
    if f(i) == 6:
        print(i)  # Результат: 728
        break
💡 Способ 2: Мемоизация через список
LIMIT = 5000
memo = [-1] * LIMIT   # список, -1 означает "не вычислено"

def f(n):
    if memo[n] != -1:   # уже вычислено
        return memo[n]

    if n == 0:
        memo[n] = 0
    elif n % 3 == 2:
        memo[n] = f(n - 1) + 1
    else:
        memo[n] = f((n - n % 3) // 3)

    return memo[n]

for i in range(LIMIT):
    if f(i) == 6:
        print(i)  # Результат: 728
        break
💡 Способ 3: Итеративное заполнение таблицы (динамическое программирование)
LIMIT = 5000
dp = [0] * LIMIT   # dp[n] = F(n)

for n in range(1, LIMIT):
    if n % 3 == 2:
        dp[n] = dp[n - 1] + 1
    else:
        dp[n] = dp[(n - n % 3) // 3]

for i in range(LIMIT):
    if dp[i] == 6:
        print(i)  # Результат: 728
        break
✅ Преимущество: Не использует рекурсию вообще. Нет риска RecursionError. Работает быстрее всех остальных способов — все значения заполняются за один проход.

📊 Сравнение всех подходов

Подход RecursionError Скорость Сложность кода Рекомендуется
Прямая рекурсия Возможен Медленная Минимальная Малый диапазон (<900)
Рекурсия + setrecursionlimit Маловероятен Медленная Минимальная Быстрое исправление ошибки
Рекурсия + dict-кэш Маловероятен Быстрая Средняя Повторяющиеся вызовы
Рекурсия + список-кэш Маловероятен Быстрая Средняя Числовые аргументы
Итеративное ДП (таблица) Невозможен Самая быстрая Средняя ✅ Большой диапазон

✅ Итоговая рекомендация:

  1. Начните с прямой рекурсии — перепишите условие задачи «один в один».
  2. Получили RecursionError? Добавьте sys.setrecursionlimit(10000).
  3. Всё ещё падает? Перейдите на итеративное динамическое программирование.
  4. Нужна максимальная скорость? Используйте таблицу ДП или @lru_cache.

🎓Заключение

Рекурсия — мощный инструмент, но требует осторожности при работе с большими значениями аргументов.


Помните:

  • Прямая рекурсия — самый читаемый код
  • sys.setrecursionlimit() — первая помощь при RecursionError
  • Мемоизация через список/словарь ускоряет повторные вызовы
  • Итеративное ДП — самое надёжное и быстрое решение
